# Death Rewind

Death Rewind 是一个围绕“死亡后快速回到安全状态”设计的 Minecraft 模组。

它会在平时自动创建回溯点，并在你死亡时提供一键回溯入口，让你直接回到几分钟前的世界状态，减少高风险探索、BOSS 战和极限玩法中的挫败感。

## 核心功能

1. 死亡界面一键回溯
- 进入死亡界面后，会出现“回溯到若干分钟前”按钮。
- 点击后会向本地回溯后端发送恢复请求。

2. 自动检查点（自动回溯点）
- 按设定间隔自动创建回溯点。
- 创建成功或失败可在游戏内提示（可关闭）。

3. 回溯期间自动处理流程
- 回溯前会先保存本地世界与玩家数据。
- 回溯过程中会冻结自动保存，避免恢复窗口内写入冲突。
- 回溯完成后会自动尝试重连世界，并在屏幕上提示结果。

4. 死亡界面“强制回溯模式”
- 可选开启。
- 开启后，死亡界面仅保留回溯按钮，其他按钮会被禁用。

5. 失败保护与稳定性机制
- 回溯请求有短冷却，避免重复点击导致多次触发。
- 自动保存冻结存在超时保护，超时后会自动恢复保存。
- 与后端版本不兼容时会给出明确提示。

## 当前支持情况

- 模组加载器：Fabric
- Minecraft 版本：1.21.x（当前实现目标为 1.21 至 1.21.10）
- Java：21+
- 依赖：Fabric API

说明：仓库中预留了 Forge / NeoForge 目录，但目前功能实现以 Fabric 版本为主。

## 使用前准备

Death Rewind 需要本地回溯后端程序配合工作。

如果后端未启动或通信失败：
- 死亡界面的回溯按钮会不可用。
- 游戏内会提示“无法连接主程序，死亡回溯功能将不可用”。

## 安装与使用

1. 安装 Fabric Loader 与 Fabric API（匹配你的游戏版本）。
2. 将 Death Rewind 模组放入 mods 文件夹。
3. 启动并保持本地回溯后端可用。
4. 正常游玩，等待自动回溯点生成。
5. 死亡后在死亡界面点击“回溯到若干分钟前”。

## 配置说明

配置文件名：`deathrewind.properties`

默认配置：

```properties
intervalMinutes=5
showBackupInfo=true
forceDeathRewind=false
```

参数解释：

- `intervalMinutes`
  - 自动创建回溯点的间隔（分钟）。
  - 有效范围 1 到 1440。

- `showBackupInfo`
  - 是否在聊天栏显示“回溯点创建成功/失败”提示。

- `forceDeathRewind`
  - 是否启用“强制回溯模式”。
  - 开启后死亡界面只允许点击回溯按钮。

## 常见问题

1. 死亡后按钮是灰色/不可点击
- 通常是后端未连接成功，或正在握手检测。
- 先确认后端是否正在运行，再重新进入世界测试。

2. 回溯后没有自动回到世界
- 模组会自动重试重连；若多次失败，会回到世界选择界面。
- 先检查世界名是否正常、后端是否正常响应，再重试。

3. 看到版本不兼容提示
- 说明当前模组版本与后端版本不满足最低兼容要求。
- 升级到彼此兼容的版本后再使用。

## 项目目标

Death Rewind 关注的是“减少死亡惩罚带来的挫败感”，而不是“取消生存压力”。

你仍然需要探索、战斗、承担风险，但当意外死亡发生时，可以有一次更平滑、更可控的回到战场方式。
